# Contributing Guidelines

Thank you for contributing to Weaver Forge.

### Core Principles
- Evidence first: Every claim must be backed by receipts.
- Build more than you talk.
- Failed experiments are welcome when documented.
- Contributor ≠ Witness — you cannot witness your own work.

### Ways to Contribute
- Daily commits and build receipts
- Work in project pods
- Code reviews and witness reproductions
- Improving templates and documentation
- Building the Daily Commit Ledger

### Submission Requirements
- Use the provided receipt and witness templates
- Link clear evidence (commits, PRs, notebooks, screenshots)
- Clearly state what your work proves and does not prove
- Follow the daily check-in format in Discord

### Questions & Discussions
Use GitHub Discussions or the appropriate Discord channels. Always include context, code, errors, and what you have already tried.

### Code of Conduct
See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
