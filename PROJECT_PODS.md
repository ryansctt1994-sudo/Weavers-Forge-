# Project Pods

Project Pods are small working groups for consistent building and mutual review.

### Purpose
- Build real projects together
- Stay accountable
- Review work and reproduce results
- Produce strong evidence

### Pod Rules
- Clear project goal + 7-day milestones
- Public progress thread / GitHub issue
- At least one weekly receipt
- Weekly pod update

### Starter Pods
- **ML Foundations Pod** (linear/logistic regression, decision trees, k-means, PCA, etc.)
- **Deep Learning Pod** (MLP, CNN, RNN/LSTM, mini-transformer, fine-tuning, etc.)
- **DSA Pod** (LeetCode, patterns, visualizers, mock interviews)
- **Weaver Systems Pod** (Daily Commit Ledger, receipt tools, claim registries, replay systems, etc.)

See `RECEIPT_TEMPLATE.md` and `WITNESS_REVIEW_TEMPLATE.md` for formats.

---

**Build. Test. Commit. Receipt. Repeat.**
